3. Architectural & Implementation Options

Select the implementation approach you want to take.

A

Shared Widget Architecture (Recommended)

Build a generic FeatureCoachMark widget in lib/core/presentation/widgets/ and wrap screens in a stack. The "Rune & Socket" is dynamic based on checking if habit.twoMinuteVersion or habit.reward etc is set.

B

Screen-by-Screen Implementation

Create separate overlay classes for each screen. This allows screen-specific custom shapes but has more boilerplate. The card indicator is a simple static edit button.
