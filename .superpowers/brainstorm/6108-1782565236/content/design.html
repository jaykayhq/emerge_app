1. Shared Data Model

Body Structure

Head — sphere
Neck — short capsule
Torso — tapered capsule (wider at shoulders)
Upper Arms — capsule
Forearms — capsule
Thighs — capsule
Shins — capsule
Hands — small sphere
Feet — rounded box

11 components, each with position, rotation, scale, color, material

Color Slots

skinColor — base body color (default by archetype)
outlineColor — rim/edge accent color
accentColor — secondary (eyes, details)
glowColor — evolution glow/aura
equipmentColors — per equipped item

Users pick from palette or custom hex — live preview in customizer

Equipment Slots

head — hats, helmets, hair, halos, horns
back — capes, wings, backpacks, aura
leftHand — weapons, tools, instruments
rightHand — weapons, shields, orbs
feet — shoes, boots, anklet effects
waist — belts, sashes
aura — full-body effect overlays

Each slot: null (empty) or ShopItem (with visual data)

Evolution Integration

Phantom — semi-transparent, faint glow
Construct — structural lines visible
Incarnate — solid, core glow appears
Radiant — kintsugi cracks, pulsing core
Ascended — sparkles, full aura flourish

Maps to your existing evolution system — applies as material/glow overrides

2. Procedural 3D Renderer (CustomPainter)

The workhorse renderer for lists, previews, and customization UI. Each body part is a capsule shape drawn on Canvas:

        ╭────────╮
        │ (⊙)   │  ← Head: circle + radial gradient
        ╰────────╯    (lighter at upper-left = fake sun)
          ║
        ╔═╩══╗    ← Torso: rounded rect with
        ║    ║      vertical gradient from light top
        ║    ║      to dark bottom
        ╚═╦══╝
       ┌─╨─┐┌─╧─┐  ← Arms: rounded rects/capsules
       │   ││   │    with shadow on bottom edge
       └───┘└───┘
        ╱ ║ ╲
       ╱  ║  ╲    ← Legs: same capsule + gradient
      ▕   ║   ▏    with ground shadow ellipse
       ▔▔▔ ▔▔▔

Shading technique

Each part uses a linear gradient from lighter (upper-left) to darker (lower-right) to simulate a fixed light source. Equipment items are drawn as additional shapes in the correct z-order.

Draw order: back limbs → back equipment → torso → head → front equipment → front limbs → aura overlay

3. ThreeFlutter 3D Renderer (Profile View)

Render Pipeline

1. Shared AvatarData → build ThreeScene 2. Create MeshStandardMaterial per body part (color, roughness, metalness) 3. Position SphereGeometry | CapsuleGeometry | CylinderGeometry at skeleton joints 4. Apply equipment meshes at attachment points 5. Set evolution glow as emissive + point light 6. Render with OrbitControls (drag to rotate)

The ThreeFlutter scene is wrapped in a UiKitView / HtmlElementView depending on platform. The model is built from primitives at runtime — no glTF files needed. Equipment items are pre-built Three.js geometries attached to skeleton joints.

4. Customization UI

Tab 1: Colors

Color picker per body region:

  • Skin / Body fill
  • Outline / rim color
  • Accent color (eyes, details)
  • Equipment tint
  • Aura / glow color

Each has archetype preset + custom hex picker. Live preview on the procedural renderer updates in real-time.

Tab 2: Equipment

Slot-by-slot grid:

  • Shows owned items per slot
  • Locked items shown grayed with unlock requirement
  • Tap to equip — preview updates immediately
  • "Visit Shop" link per slot

Tab 3: Evolution

Read-only display:

  • Current phase + progress to next
  • Unlocked body artifacts with visual preview
  • Preview of next unlock

Tab 4: Profile View

Full 3D showcase:

  • Rendered via ThreeFlutter
  • 360° drag-to-rotate
  • Pinch to zoom
  • Auto-rotation toggle
  • Pose selector buttons

5. Shop Integration

ShopItem {
id: "crystal_crown",
name: "Crystal Crown",
slot: EquipmentSlot.head,
rarity: Rarity.epic,
priceXP: 2000,
priceGems: null,
procedural: { drawFn: "Crown", params: {...} },
threeDAsset: { type: "primitive", data: {...} }
}

Each shop item provides draw instructions for both renderers — procedural (simple shape + gradient) and 3D (primitive geometry). New items can be added server-side via Firestore.
