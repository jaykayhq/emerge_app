

## Command & Test Execution Rules
- Do not run multiple commands at once.
- Do not run full test (lutter test) or full analyze (dart analyze) until ALL work is completely finished.
- Double the test timeout time when running tests (e.g. --timeout 60s).
- Only run tests on the specific file you are working on, not the whole suite.
- Do not be impatient: commands must finish or fail before you do anything else.
